ProxySQL Universal
REST API

A single Flask service that wraps both ProxySQL admin interfaces — MySQL (port 6032) and PostgreSQL (port 6132) — exposing clean HTTP endpoints for full server, user, and configuration management. No direct console access needed.

Installation setup

01

Install dependencies

bash
pip install flask pymysql psycopg2-binary
02

Deploy the API

bash
sudo mkdir -p /opt/proxysql-api
sudo cp app.py /opt/proxysql-api/
03

Install systemd service & start

bash
sudo cp proxysql-api.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable --now proxysql-api
sudo systemctl status proxysql-api
04

Verify it's running

bash
curl http://127.0.0.1:8080/health

Configuration env vars

Set these environment variables in the systemd service file /etc/systemd/system/proxysql-api.service:

MySQL — Servers MySQL

GET/mysql/servers
List all MySQL backend servers (runtime view).
bash
curl http://127.0.0.1:8080/mysql/servers
POST/mysql/servers
Add a MySQL backend server. Changes are immediately loaded to runtime and saved to disk.
bash
curl -X POST http://127.0.0.1:8080/mysql/servers \
  -H "Content-Type: application/json" \
  -d '{
    "hostgroup_id": 10,
    "hostname": "192.168.122.199",
    "port": 3306,
    "use_ssl": 1,
    "weight": 1,
    "max_connections": 1000,
    "status": "ONLINE",
    "comment": "MySQL Server 1"
  }'
FieldTypeRequiredDefaultNotes
hostgroup_idintrequired–Routing hostgroup (e.g. 10=write, 20=read)
hostnamestringrequired–IP or DNS of MySQL server
portintoptional3306MySQL port
use_sslintoptional01 = enforce SSL
weightintoptional1Load balancing weight
max_connectionsintoptional1000Max backend connections
max_replication_lagintoptional0Max allowed replication lag (seconds)
statusstringoptionalONLINEONLINE / SHUNNED / OFFLINE_SOFT / OFFLINE_HARD
commentstringoptional""Free-text label
PUT/mysql/servers
Update an existing server. Requires hostname + hostgroup_id to identify it.
bash
curl -X PUT http://127.0.0.1:8080/mysql/servers \
  -H "Content-Type: application/json" \
  -d '{"hostgroup_id":10,"hostname":"192.168.122.199","status":"OFFLINE_SOFT"}'
DELETE/mysql/servers
Remove a server from a hostgroup.
bash
curl -X DELETE http://127.0.0.1:8080/mysql/servers \
  -H "Content-Type: application/json" \
  -d '{"hostgroup_id":10,"hostname":"192.168.122.199"}'

MySQL — Query Rules MySQL

Route queries to different hostgroups based on pattern matching or username.

POST/mysql/query_rules
Route all SELECT queries to a read hostgroup.
bash
curl -X POST http://127.0.0.1:8080/mysql/query_rules \
  -H "Content-Type: application/json" \
  -d '{
    "match_pattern": "^SELECT",
    "destination_hostgroup": 20,
    "apply": 1,
    "comment": "Route SELECTs to readers"
  }'
DELETE/mysql/query_rules/<rule_id>
bash
curl -X DELETE http://127.0.0.1:8080/mysql/query_rules/1

MySQL — Replication Hostgroups MySQL

Define writer/reader hostgroup pairs for automatic replication-aware routing.

POST/mysql/replication_hostgroups
bash
curl -X POST http://127.0.0.1:8080/mysql/replication_hostgroups \
  -H "Content-Type: application/json" \
  -d '{"writer_hostgroup":10,"reader_hostgroup":20,"comment":"Primary/Replica"}'

PostgreSQL — Monitor Config PostgreSQL

Configure the credentials and intervals ProxySQL uses to health-check PostgreSQL backends. The monitor user must already exist on your PostgreSQL servers.

PUT/pgsql/monitor
bash
curl -X PUT http://127.0.0.1:8080/pgsql/monitor \
  -H "Content-Type: application/json" \
  -d '{
    "monitor_username": "monitor",
    "monitor_password": "monitor",
    "connect_interval": 2000,
    "ping_interval": 2000
  }'
