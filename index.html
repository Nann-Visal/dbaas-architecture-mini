HAProxy Config
REST API

A lightweight Flask service that dynamically manages HAProxy TCP frontend/backend configs per port. Create and delete per-tenant MySQL proxy configs via HTTP — HAProxy reloads instantly via admin socket with zero downtime.

Default Port :8080
Config Dir /run/haproxy/tenants
Reload Method Admin Socket
Backend Port MySQL :3306

Installation

setup
01
Create system user & directories
bash
sudo useradd -r -s /bin/false proxysql-api
sudo mkdir -p /opt/proxysql-api /etc/proxysql-api /var/log/proxysql-api
sudo mkdir -p /run/haproxy/tenants
sudo chown proxysql-api:haproxy /run/haproxy/tenants
sudo chmod 775 /run/haproxy/tenants
02
Deploy app & install dependencies
bash
sudo cp app.py /opt/proxysql-api/app.py
sudo chown -R proxysql-api:proxysql-api /opt/proxysql-api /var/log/proxysql-api
cd /opt/proxysql-api
sudo -u proxysql-api python3 -m venv venv
sudo -u proxysql-api venv/bin/pip install flask
sudo apt-get install -y socat
03
Add proxysql-api to haproxy group (for socket access)
bash
sudo usermod -aG haproxy proxysql-api
sudo usermod -aG haproxy proxyer
04
Persist tenants dir across reboots
bash
echo 'd /run/haproxy/tenants 0775 proxysql-api haproxy -' \
  | sudo tee /etc/tmpfiles.d/haproxy-tenants.conf
ℹ /run is a tmpfs and resets on reboot. This rule recreates the directory automatically on every boot.
05
Point HAProxy at the tenants directory
/etc/default/haproxy
CONFIG="/etc/haproxy/haproxy.cfg -f /run/haproxy/tenants"
06
Create environment file
bash
sudo tee /etc/proxysql-api/env > /dev/null <<EOF
API_PORT=8080
API_HOST=0.0.0.0
PYTHONUNBUFFERED=1
EOF
sudo chmod 600 /etc/proxysql-api/env
sudo chown proxysql-api:proxysql-api /etc/proxysql-api/env
07
Create & start systemd service
bash
sudo tee /etc/systemd/system/proxysql-api.service > /dev/null <<EOF
[Unit]
Description=HAProxy Config REST API
After=network.target haproxy.service

[Service]
Type=simple
User=proxysql-api
Group=proxysql-api
WorkingDirectory=/opt/proxysql-api
EnvironmentFile=/etc/proxysql-api/env
ExecStart=/opt/proxysql-api/venv/bin/python /opt/proxysql-api/app.py
Restart=always
RestartSec=5
NoNewPrivileges=true
PrivateTmp=true

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl daemon-reload
sudo systemctl enable --now proxysql-api
sudo systemctl status proxysql-api
08
Verify it's running
bash
curl http://127.0.0.1:8080/health

Create Config

POST /haproxy/config — Create a TCP proxy config for a port

Creates a HAProxy frontend+backend config file that proxies the given port to node_ip:3306. HAProxy reloads instantly — the port starts listening with zero downtime.

bash
curl -X POST http://127.0.0.1:8080/haproxy/config \
  -H "Content-Type: application/json" \
  -d '{
    "port": 25010,
    "node_ip": "10.10.107.182"
  }'
FieldTypeRequiredDescription
port integer REQUIRED TCP port HAProxy will listen on (1–65535). Must be unique per tenant.
node_ip string REQUIRED IP address or hostname of the MySQL backend server. HAProxy proxies to this host on port 3306.
200 success
{
  "status": "ok",
  "message": "HAProxy config for port 25010 created and loaded"
}
500 error
{
  "status": "error",
  "message": "HAProxy reload failed, rolled back..."
}
ℹ If HAProxy reload fails after writing the config, the file is automatically rolled back to its previous state (or deleted if it was new). The API never leaves HAProxy in a broken state.
Generated config file — /run/haproxy/tenants/25010.cfg
haproxy config
frontend mysql_front_25010
    bind *:25010
    mode tcp
    option tcplog
    default_backend sql_user_25010

backend sql_user_25010
    mode tcp
    server mysql1 10.10.107.182:3306 check

Delete Config

DELETE /haproxy/config — Remove a TCP proxy config by port

Deletes the config file for the given port and reloads HAProxy. The port immediately stops accepting connections. If reload fails, the file is restored automatically.

bash
curl -X DELETE http://127.0.0.1:8080/haproxy/config \
  -H "Content-Type: application/json" \
  -d '{"port": 25010}'
FieldTypeRequiredDescription
port integer REQUIRED The port number of the config to remove.
200 success
{
  "status": "ok",
  "message": "HAProxy config for port 25010 deleted and loaded"
}
404 not found
{
  "status": "error",
  "message": "No config found for port 25010"
}
